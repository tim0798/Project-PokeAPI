Acerca de MiPokemonAPI

Esta es una API de servicios REST que su funcionalidad parte de las especificaciones del documento Test Java.

Servicios REST

Generaciones Obtener los Pokémones de cada generación (Consumiendo la API https://pokeapi.co/ desde Spring Boot)
GET http://localhost:8080/MiPokemonAPI/generaciones/{generacion}

Parámetros Tipo Requerido Posibles valores
{generacion} String Si 1, 2, 3, 4, 5 y 6
Respuesta ejemplo:
[{"id":1,"name":"bulbasaur","types":[{"slot":1,"type":{"name":"grass","url":"https://pokeapi.co/api/v2/type/12/"}},{"slot":2,"type":{"name":"poison","url":"https://pokeapi.co/api/v2/type/4/"}}]},{"id":2,"name":"ivysaur","types":[{"slot":1,"type":{"name":"grass","url":"https://pokeapi.co/api/v2/type/12/"}},{"slot":2,"type":{"name":"poison","url":"https://pokeapi.co/api/v2/type/4/"}}]}]

Rango de Pokémones Obtener una lista de Pokémon con un rango que va del Pokémon número a al b (Consumiendo la API https://pokeapi.co/ desde Spring Boot)
GET http://localhost:8080/MiPokemonAPI/listaPokemones/{a}/{b}

Parámetros Tipo Requerido Posibles valores
{a} Entero Si 1, 2, 3, 4, 5, etc.
{b} Entero Si 2, 4, 6, 8, 10, etc.
Respuesta ejemplo:
[{"id":11,"name":"metapod","types":[{"slot":1,"type":{"name":"bug","url":"https://pokeapi.co/api/v2/type/7/"}}]},{"id":12,"name":"butterfree","types":[{"slot":1,"type":{"name":"bug","url":"https://pokeapi.co/api/v2/type/7/"}},{"slot":2,"type":{"name":"flying","url":"https://pokeapi.co/api/v2/type/3/"}}]},{"id":13,"name":"weedle","types":[{"slot":1,"type":{"name":"bug","url":"https://pokeapi.co/api/v2/type/7/"}},{"slot":2,"type":{"name":"poison","url":"https://pokeapi.co/api/v2/type/4/"}}]}]

Obtener Pokémon Obtener Pokémon buscándolo por su nombre o número (Consumiendo la API https://pokeapi.co/ desde Spring Boot)
GET http://localhost:8080/MiPokemonAPI/buscarPokemon/{identificador}

Parámetros Tipo Requerido Posibles valores
{identificador} String, Entero Si String {golbat, oddish, venonat, wigglytuff, etc.}, Enteros {1, 2, 3, 4, 5, 6, etc.}
Respuesta ejemplo:
{"id":43,"name":"oddish","types":[{"slot":1,"type":{"name":"grass","url":"https://pokeapi.co/api/v2/type/12/"}},{"slot":2,"type":{"name":"poison","url":"https://pokeapi.co/api/v2/type/4/"}}]}

Guardar Pokémones Local Almacenar un Pokémon en la base de datos local (h2), guardando un apodo único para él, su especie, su ataque, su defensa y su salud (Estos tres últimos la aplicación las genera al azar en un rango de 0 a 15 con números enteros).
PUT http://localhost:8080/MiPokemonAPI/guardarPokemon

Parámetros Tipo Requerido Posibles valores
{apodo} String Si Pokemono, Pikachuy, Pikoro, etc.
{especie} String Si Aves, Reptiles, Ranas, etc.
Respuesta ejemplo:
¡Pokemon guardado! PokemonEntity [id=6, apodo=Rana Cola Pelada, especie=Reptiles, ataque=9, defensa=2, salud=14]

Obtener Pokémon LocalConsultar desde la base de datos local (h2) un Pokémon capturado por su apodo
GET localhost:8080/MiPokemonAPI/obtenerPokemon/{apodo}

Parámetros Tipo Requerido Posibles valores
{apodo} String Si Pokebola, Pokegato, Pokerana, etc.
Respuesta ejemplo:
{"id":9,"apodo":"Pokegato","especie":"Felino","ataque":4,"defensa":1,"salud":0}

Pokémones Especies LocalConsultar desde la base de datos local (H2) todos los Pokémon capturados de una misma especie.
GET http://localhost:8080/MiPokemonAPI/consultarPokemonesPorEspecie/{especie}

Parámetros Tipo Requerido Posibles valores
{especie} String Si Felinios, Ovíparo, Aves, Reptiles, etc
Respuesta ejemplo:
[{"id":9,"apodo":"Pokegato","especie":"Felino","ataque":4,"defensa":1,"salud":0},{"id":10,"apodo":"Tigre","especie":"Felino","ataque":12,"defensa":4,"salud":0},{"id":11,"apodo":"Chita","especie":"Felino","ataque":3,"defensa":11,"salud":10}]

Test Java con respuestas

  1. Usando los servicios ofrecidos en PokeApi (https://pokeapi.co/) desarrollar una aplicación standalone con Java 8, Maven, Base de datos H2 y el Framework de Spring Boot que exponga servicios web REST que ofrezca las siguientes funcionalidades:
    1. Obtener una lista con los Pokémon que componen las generaciones de la 1ra a la 6ta, en esa lista debe aparecer el nombre de los Pokémon, su número y sus tipos.
      1. 1ra generación: Pokémon del 1 al 151. GET http://localhost:8080/MiPokemonAPI/generaciones/1
      2. 2da generación: Pokémon del 152 al 251. GET http://localhost:8080/MiPokemonAPI/generaciones/2
      3. 3ra generación: Pokémon del 252 al 386. GET http://localhost:8080/MiPokemonAPI/generaciones/3
      4. 4ta generación: Pokémon del 387 al 493. GET http://localhost:8080/MiPokemonAPI/generaciones/4
      5. 5ta generación: Pokémon del 494 al 649. GET http://localhost:8080/MiPokemonAPI/generaciones/5
      6. 6ta generación: Pokémon del 650 al 721. GET http://localhost:8080/MiPokemonAPI/generaciones/6
    2. Obtener una lista de Pokémon con un rango que vaya del Pokémon número a al b, en esa lista debe aparecer el nombre de los Pokémon, su número y sus tipos. GET http://localhost:8080/MiPokemonAPI/listaPokemones/2/6
    3. Obtener el nombre, número y tipos de un Pokemon buscándolo por su nombre o número. GET http://localhost:8080/MiPokemonAPI/buscarPokemon/ivysaur
    4. Almacenar un Pokémon en la base de datos, guardando un apodo para él, debe ser único, su especie, su ataque, su defensa y su salud (Estos tres últimos la aplicación debe general estos valores al azar en un rango de 0 a 15 con números enteros). PUT http://localhost:8080/MiPokemonAPI/guardarPokemon
    5. Consultar desde la base de datos un Pokémon capturado por su apodo, debe devolver el apodo, la especie, ataque, defensa, salud, tipos y número. GET http://localhost:8080/MiPokemonAPI/obtenerPokemon/pokebola
    6. Consultar desde la base de datos todos los Pokémon capturados de una misma especie. GET http://localhost:8080/MiPokemonAPI/consultarPokemonesPorEspecie/Reptiles
    7. Servicio en la url /help que te devuelva toda la información necesaria para consumir los otros 6, el nombre del desarrollador y su información de contacto. En vez de un servicio REST este se puede desarrollar como una web simple solo con HTML5, CSS3 y JS estándar. GET http://localhost:8080/MiPokemonAPIMiPokemonAPI/help
  2. Entregar el código del proyecto en un repositorio público de GitHub o GitLab, con un archivo README.md que como compilarlo y ejecutarlo. Se puede agregar al repositorio el ejecutable, es opcional. Se toma en cuenta manejo de LOGs, base de datos, consumo de servicios de PokeApi, documentación técnica (Javadoc y README.md), uso de los verbos HTTP para consumir los servicios REST y uso de los estados HTTP para responder a estos.
